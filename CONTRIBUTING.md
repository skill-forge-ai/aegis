# Contributing to Aegis

Thank you for your interest in contributing to Aegis!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest features
- Include clear steps to reproduce for bugs
- For feature requests, describe the problem you're trying to solve

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Ensure all templates are valid and scripts are tested
5. Submit a PR with a clear description

### Development Setup

```bash
git clone https://github.com/skillforge-ai/aegis.git
cd aegis
```

No build step required — Aegis is a collection of templates, scripts, and documentation.

### Guidelines

- **Templates** should be generic and adaptable to different tech stacks
- **Scripts** should work on macOS and Linux (bash 4+)
- **Documentation** should be clear and actionable
- Follow existing file structure and naming conventions

### Areas We'd Love Help With

- PM tool adapters (Linear, Notion, GitHub Projects)
- CI pipeline templates for different platforms
- Language-specific contract test examples
- Translations of documentation

## Code of Conduct

Be respectful, be constructive, be kind. We're all here to build better software.
